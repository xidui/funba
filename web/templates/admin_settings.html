Pipeline Publishing

Per-pipeline-per-platform configuration. Generate = create a variant for that platform when the pipeline runs. Auto-publish = publish without human review. For platforms with external APIs (Twitter, Hupu, …) auto-publish kicks off the publisher script on creation. For Funba (home feed) auto-publish marks the delivery published and the post appears on the home waterfall immediately.

Metric Pipeline Flags

Legacy game metric fan-out
Default off. Current production metrics are season-triggered only. Enable this only if you intentionally want `ingest_game` to revive the deprecated per-game `compute_game_delta -> reduce_after_ingest` path for legacy experiments.

Ingest → Metric Blocking

Control which data artifacts must complete before metrics are triggered. Non-blocking artifacts are backfilled after metrics fire (won't delay metric results).

Shot Chart
When ON, shot chart backfill must finish before metrics fire (can take hours). When OFF, metrics fire immediately after detail+PBP; shot charts backfill in the background.
Line Score
When ON, line score backfill must finish before metrics fire. When OFF (default), line score is fetched after metrics.
Period Stats
When ON, per-quarter player stats must finish before metrics fire. When OFF (default), period stats are fetched after metrics.

Hero Card Poster (gpt-image-2)

Edits take effect on the next curator run for any game; you can also regenerate retroactively for one game below.
